## 端口统一

| 原服务 | 原端口 | 新路径 | 说明 |
|--------|--------|--------|------|
| TTS服务 | 8880 | `/tts/*` | 语音合成 |
| 实时语音 | 8881 | `/ws/voice/*` | WebSocket |
| 代码解释器 | 8882 | `/code/*` | Python执行 |
| RAG服务 | 8000 | `/rag/*` | 向量数据库 |

统一后：**端口 8888**

## API端点

### TTS服务
- `POST /tts` - 文本转语音
- `GET /characters` - 获取可用音色
- `GET /audio/{filename}` - 获取音频文件
- `POST /tts/qwen` - 千问TTS

### 代码解释器
- `POST /code/execute` - 执行Python代码
- `POST /code/reset` - 重置执行会话

### RAG向量数据库
- `POST /rag/upload` - 上传文档
- `POST /rag/search` - 搜索文档
- `GET /rag/stats` - 获取统计
- `DELETE /rag/documents` - 删除文档

### 实时语音
- `WS /ws/voice/{client_id}` - WebSocket连接

### 通用
- `GET /health` - 健康检查
- `GET /docs` - API文档 (Swagger UI)

## 启动方式

### 启动统一服务
```bash
cd my-chat-ui/sovits-service
start_unified.bat
# 或
python unified_server.py
```

### 前端开发
```bash
cd my-chat-ui/apps/web
pnpm dev
```

## 优势

1. **简化部署** - 只需启动一个服务
2. **统一管理** - 共享配置和日志
3. **降低维护成本** - 单一代码库
4. **性能优化** - 避免跨服务通信开销
5. **开发友好** - 单一调试入口

## 迁移说明

如果你之前使用多个服务，迁移步骤：

1. 停止旧服务（端口 8880, 8881, 8882, 8000）
2. 启动统一服务（端口 8888）
3. 前端会自动通过代理访问新端口
4. 所有功能保持不变
